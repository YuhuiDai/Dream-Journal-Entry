      Final Project:

      The web displays your dream journal. A user can build his or her own dream box by signing in and create a name. Besides, the user can check out other's dream journal from the home page and comment on it. When you write your dream down, the sentiment analysis api can tell you whether it is a good dream or bad dream or neutral dream.

      Why I do it:

      I always have pretty vivid dream and sometimes amazingly interesting dream. I want to keep a dream journal so that I can write down what I dreamt so that I would not forget. Sometimes the dream is so complex, with a lot of postive and negative emotion involved. I am wondering whether it is a good sign or a bad sign. In my application, I used a natural language processing api to analyze the sentiment in my text. With the final algorithm offered by the api, I can have a sense of the overall emotional tendency.

      How the code works:

    1. Home page displays all the dream journals, and their related comments
    2. User can click the comment button to comment on the dream and save it
    3. Login page allows user to either create a new page or entering their existing page by type in their user name
    4. Based on which button the user click, call different functions
    5. On the individual user page, one can write his or her dream in text
    6. The api will tell you immediately whether it is a good dream or a bad dream
    7. Then, the page automatically add your dream to your previous dream journals if they exist

      Challenges:

    1. The navigating from one page to another page is tricky at first but once I understand the mechanism, it is much easier
    2. Displaying dream journals' date is difficult at first because the way the data is displayed is different when it is stored freshly and when it is retrived from the database. So I have to create two different functions based on the in which way the data are displayed, either retrieval or just created
    3. The asynchronized issue happened frequently when I am coding. I have to be extra careful about where I put it
    4. I have to create two type of objects in my database, the dreams and the comments and match them together. By associating their id together, I know where to put which comment
    5. At first, when I click on comment, suddenly all the section under comments of each dreams are open for me to edit. I only want to comment on the one I click. I have to create a unique id for that specifc section in html. The way to address the issue is by using the id of that particular dream as the base id to give the comment section a unique id that relates to the dream id. In this way, I can make sure that every time someone clicks on comment button, he or she only can comment on the specific dream, not everything
